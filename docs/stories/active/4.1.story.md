# Story 4.1: Upload e Gestão de Anexos

**Epic:** 4 — Assistente Clínico e Documentos
**Status:** Draft
**Prioridade:** P0
**FR:** FR-11, FR-20, FR-23
**Estimativa:** L (6-8h)

## Descrição

Como farmacêutico, preciso anexar fotos e PDFs ao prontuário do paciente (receitas médicas, exames laboratoriais, diário glicêmico, MRPA, organizadores de medicamentos), com visualização inline e limite de armazenamento de ~10 páginas por paciente.

## Acceptance Criteria

- [ ] AC-1: Upload de arquivos: PDF e imagens (JPG, PNG)
- [ ] AC-2: Tipos de documento para categorização: Receita Médica, Exame Laboratorial, Diário Glicêmico, MRPA, Organizador de Medicamentos, Outros
- [ ] AC-3: Limite de armazenamento: máximo 10 arquivos ou ~20MB por paciente (configurável)
- [ ] AC-4: Visualização inline de imagens no prontuário; link para download de PDFs
- [ ] AC-5: Cada anexo tem: nome, tipo, data de upload, tamanho, descrição opcional
- [ ] AC-6: Exclusão de anexo com confirmação
- [ ] AC-7: Alerta ao farmacêutico quando paciente se aproximar do limite (80%)

## Escopo

**IN:**
- Upload para sistema de arquivos local ou object storage (configurável)
- Limite por paciente
- Listagem e visualização no prontuário

**OUT:**
- OCR automático de receitas (v2)
- Integração com Supabase Storage ou S3 (configurável via .env)

## Dependências

- Story 1.1 (paciente)

## Decisões Técnicas

- Armazenamento: `/public/uploads/pacientes/{id}/` (MVP) — pode ser migrado para S3/Supabase
- Limite: configurável via env `STORAGE_LIMIT_PER_PATIENT_MB` (default: 20MB)

## Tasks de Implementação

- [ ] Criar model `Anexo` no Prisma: id, pacienteId, nome, tipo (enum), caminhoArquivo, tamanhoBytes, descricao, criadoEm
- [ ] Criar migration
- [ ] Criar API `/api/anexos` com multipart/form-data upload
- [ ] Validação de tipo de arquivo e tamanho no backend
- [ ] Verificação de limite por paciente antes de aceitar upload
- [ ] Criar endpoint de servir arquivo: `/api/anexos/[id]/arquivo`
- [ ] UI de upload com drag-and-drop ou seleção de arquivo
- [ ] Galeria de anexos no prontuário com preview de imagens
- [ ] Barra de uso de armazenamento por paciente
