# Story 2.4: Registro de PRM/PRF

**Epic:** 2 — Farmacoterapia
**Status:** Draft
**Prioridade:** P0
**FR:** FR-12, FR-21
**Estimativa:** M (4h)

## Descrição

Como farmacêutico, preciso identificar, classificar e registrar os Problemas Relacionados a Medicamentos (PRM) e Problemas Relacionados à Farmacoterapia (PRF) do paciente, para documentar formalmente os achados clínicos e direcionar as intervenções.

## Acceptance Criteria

- [ ] AC-1: Categorias de PRM disponíveis para seleção:
  - Necessidade (medicamento desnecessário, medicamento necessário não prescrito)
  - Efetividade (dose inadequada, medicamento ineficaz)
  - Segurança (dose acima do necessário, reação adversa)
  - Adesão (baixa adesão intencional / não intencional) — se taxa objetiva disponível via Story 2.5 (contagem de comprimidos), o valor é exibido automaticamente como evidência ao selecionar esta categoria
- [ ] AC-2: Para cada PRM: categoria, descrição detalhada, medicamento relacionado (opcional), gravidade (Leve/Moderado/Grave)
- [ ] AC-3: Múltiplos PRMs por atendimento
- [ ] AC-4: Status do PRM: Identificado / Em Acompanhamento / Resolvido
- [ ] AC-5: PRMs exibidos no prontuário do paciente com histórico de resolução
- [ ] AC-6: PRMs podem ser usados como contexto para geração de intervenções (Story 3.1)

## Escopo

**IN:**
- Model `ProblemaMedicamento` no Prisma
- CRUD completo com categorização
- Listagem por paciente com filtro de status

**OUT:**
- Integração com sistema de classificação PCNE (v2)
- Geração automática de PRM via IA (coberto na Story 4.2)
