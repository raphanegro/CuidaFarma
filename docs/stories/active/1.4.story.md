# Story 1.4: Módulo Histórico Clínico

**Epic:** 1 — Prontuário Farmacêutico
**Status:** Done
**Prioridade:** P0
**FR:** FR-5
**Estimativa:** M (3-4h)

## Descrição

Como farmacêutico, preciso registrar as doenças diagnosticadas e condições clínicas relevantes do paciente (CID, comorbidades, condições crônicas), para contextualizar a análise farmacoterapêutica.

## Acceptance Criteria

- [x] AC-1: Registro de doenças diagnosticadas com: nome da doença, CID-10 (opcional), data do diagnóstico (opcional)
- [x] AC-2: Campo para outras condições clínicas relevantes (texto livre)
- [x] AC-3: Múltiplas doenças por paciente
- [x] AC-4: Possibilidade de marcar doença como "ativa" ou "resolvida"
- [x] AC-5: Histórico clínico exibido no prontuário do paciente
- [x] AC-6: Condições clínicas ordenadas por prioridade/data

## Escopo

**IN:**
- Model `HistoricoClinico` associado ao paciente
- CRUD de doenças/condições
- Seção "Histórico Clínico" no prontuário

**OUT:**
- Lookup automático de CID-10 via API externa (v2)
- Integração com sistema de prontuário eletrônico

## Dependências

- Story 1.1 (modelo Paciente expandido)

## Valor de Negócio

Permite ao farmacêutico identificar comorbidades que interferem diretamente na farmacoterapia. É pré-requisito direto para a Análise Farmacoterapêutica (Story 2.3) e para a Estratificação de Risco (Story 3.3). Sem este módulo, a IA clínica (Story 4.2) não tem contexto de doenças para gerar alertas relevantes.

## Riscos

- **R1 (Baixo):** CID-10 tem ~70k códigos — busca/autocomplete completo está fora de escopo no MVP. Mitigação: campo texto livre para doença + campo opcional de CID-10 manual.
- **R2 (Baixo):** Doenças marcadas como "resolvidas" mas ainda relevantes para o histórico. Mitigação: nunca deletar — apenas mudar status para RESOLVIDA com data de resolução.

## Definition of Done

- [ ] Farmacêutico pode adicionar doença com nome, CID-10 opcional e data de diagnóstico
- [ ] Doenças podem ser marcadas como ATIVA ou RESOLVIDA
- [ ] Múltiplas condições por paciente suportadas
- [ ] Histórico exibido no prontuário ordenado por data (mais recente primeiro)
- [ ] CRUD completo via API `/api/historico-clinico`
