# Story 1.5: Módulo Dados Clínicos

**Epic:** 1 — Prontuário Farmacêutico
**Status:** Done
**Prioridade:** P0
**FR:** FR-6
**Estimativa:** M (3-4h)

## Descrição

Como farmacêutico, preciso registrar os dados clínicos aferidos na consulta (peso, altura, IMC calculado, pressão arterial, frequência cardíaca, glicemia capilar), para monitorar a evolução clínica do paciente ao longo das consultas.

## Acceptance Criteria

- [x] AC-1: Campos para: peso (kg), altura (cm), IMC (calculado automaticamente), pressão arterial sistólica e diastólica (mmHg), frequência cardíaca (bpm), glicemia capilar (mg/dL)
- [x] AC-2: IMC calculado automaticamente ao preencher peso e altura, com classificação (Abaixo do peso / Normal / Sobrepeso / Obesidade I/II/III)
- [x] AC-3: Dados clínicos associados ao atendimento (timestamp específico)
- [ ] AC-4: Histórico de dados clínicos exibido em gráfico de evolução por paciente (UI — Epic 2)
- [x] AC-5: Campos opcionais (nem todos precisam ser preenchidos em toda consulta)
- [ ] AC-6: Alertas visuais para valores fora da faixa de referência (UI — próxima iteração)

## Escopo

**IN:**
- Model `DadosClinicos` por atendimento
- Cálculo automático de IMC
- Exibição de histórico com trend
- Alertas de referência (PA > 140/90, glicemia > 180, etc.)

**OUT:**
- Gráficos interativos avançados (v2)
- Integração com dispositivos médicos (v3)

## Valor de Negócio

Permite monitoramento longitudinal dos parâmetros vitais do paciente ao longo das consultas, sendo dado de entrada crítico para a Estratificação de Risco (Story 3.3) e para os Alertas Clínicos da IA (Story 4.2). Agrega valor imediato ao farmacêutico ao exibir evolução de PA, glicemia e peso em uma única tela.

## Riscos

- **R1 (Médio):** Faixas de referência clínica devem ser baseadas em guidelines validados. Mitigação: usar valores OMS/SBC amplamente aceitos — PA ≥140/90 = alerta, glicemia capilar >180 mg/dL pós-prandial = alerta, IMC via fórmula OMS (peso kg / altura m²).
- **R2 (Baixo):** Campos numéricos mal validados podem aceitar valores impossíveis (ex: PA 999/999). Mitigação: validação de range no backend e frontend com limites fisiológicos razoáveis.

## Definition of Done

- [ ] Dados clínicos (peso, altura, PA, FC, glicemia) registrados por atendimento
- [ ] IMC calculado automaticamente com classificação OMS exibida
- [ ] Alertas visuais para valores fora da faixa de referência (cor vermelha/amarela)
- [ ] Histórico tabular de dados clínicos por paciente exibido no prontuário
- [ ] Todos os campos são opcionais — form salva mesmo com preenchimento parcial
