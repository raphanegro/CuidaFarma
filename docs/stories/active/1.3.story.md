# Story 1.3: Módulo Motivo da Consulta

**Epic:** 1 — Prontuário Farmacêutico
**Status:** Done
**Prioridade:** P0
**FR:** FR-4
**Estimativa:** S (2h)

## Descrição

Como farmacêutico, preciso registrar o motivo da consulta usando opções pré-definidas e/ou descrição livre, para documentar adequadamente a razão do atendimento.

## Acceptance Criteria

- [x] AC-1: Lista de motivos pré-definidos (ex: Início de tratamento, Revisão de medicamentos, Queixa de efeito adverso, Monitoramento, Adesão, Outros)
- [x] AC-2: Campo texto livre para descrição detalhada do motivo
- [x] AC-3: Múltiplos motivos podem ser selecionados
- [x] AC-4: Motivo associado ao atendimento (não ao paciente)
- [x] AC-5: Motivo exibido no resumo do atendimento

## Escopo

**IN:**
- Enum ou tabela de motivos pré-definidos
- Campo texto complementar
- Associação ao model `Atendimento`

**OUT:**
- Motivos customizáveis pelo admin (v2)

## Dependências

- Story 1.2 (model Atendimento)

## Tasks de Implementação

- [ ] Adicionar campos `motivoConsulta` (array de enum) e `motivoDescricao` (texto) ao model `Atendimento`
- [ ] Criar migration
- [ ] Atualizar API de atendimentos para incluir motivo
- [ ] Adicionar seção de motivo no form de novo atendimento com checkboxes + textarea

## File List

- `prisma/schema.prisma`

## Valor de Negócio

Documenta formalmente o motivo de cada atendimento farmacêutico, habilitando análises epidemiológicas futuras (quais são os motivos mais frequentes?) e métricas de qualidade do serviço. Dado simples com alto retorno analítico no Epic 5.

## Riscos

- **R1 (Baixo):** Lista de motivos pré-definidos pode ser insuficiente para contextos clínicos específicos. Mitigação: opção "Outros" com campo texto livre obrigatório garante cobertura de 100% dos casos.
- **R2 (Baixo):** Múltipla seleção de motivos pode ser desconfortável na UI mobile. Mitigação: usar checkboxes com layout grid 2 colunas.

## Definition of Done

- [ ] Lista de motivos pré-definidos exibida com seleção múltipla
- [ ] Campo texto livre disponível para descrição detalhada
- [ ] Motivo(s) salvos e exibidos no resumo do atendimento
- [ ] Opção "Outros" sempre disponível e funcional
