# Story 5.2: Filtro de Busca Avançado

**Epic:** 5 — Relatórios e Gestão
**Status:** Draft
**Prioridade:** P0
**FR:** FR-29
**Estimativa:** S (2-3h)

## Descrição

Como farmacêutico, preciso buscar pacientes e atendimentos por múltiplos critérios (nome, CPF, medicamento, patologia, nível de risco) para localizar rapidamente informações clínicas.

## Acceptance Criteria

- [ ] AC-1: Busca de pacientes por: nome, CPF, unidade de saúde
- [ ] AC-2: Filtro por: nível de risco, doença/patologia, medicamento em uso
- [ ] AC-3: Resultados exibidos em tempo real (debounce de 300ms)
- [ ] AC-4: Busca global no header do dashboard (omnisearch)
- [ ] AC-5: Histórico de buscas recentes (últimas 5, armazenado localmente)

## Escopo

**IN:**
- Busca na listagem de pacientes existente (expandida)
- Omnisearch no header
- Filtros combináveis

**OUT:**
- Busca full-text com Elasticsearch (v2)

## Dependências

- Story 1.1 (dados completos do paciente)
- Módulos de Epic 2 (medicamentos, patologias)
